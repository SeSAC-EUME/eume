## 1. 제품 개요

### 1.1 제품명
이음이(EUME) - 서울시의 고립은둔청년을 위한 정서 돌봄 AI 서비스

### 1.2 제품 비전
"먼저 말을 걸고, 감정을 읽어 복지를 연결하는 AI Digital Mate"

### 1.3 목표 및 목적

#### 1.3.1 주요 목표
- **사회적 고립 해소**: 은둔·고립 청년들에게 일상적 대화와 정서적 연결 제공
- **예방적 개입**: 우울감과 무력감을 조기 감지하여 선제적 대응
- **복지 연계 강화**: AI 기반 맞춤형 복지 정보 제공 및 전문가 연결
- **지속가능한 돌봄 생태계**: 청년-전문가-기관이 협력하는 통합 플랫폼 구축

#### 1.3.2 핵심 가치 제안
- AI가 주도적으로 대화를 시작하는 능동적 케어 시스템
- 감정 데이터 기반 실시간 위험도 평가 (Emotion Score)
- 서울시 복지 정책과 실시간 연동되는 맞춤형 정보 제공
- 관리자 대시보드를 통한 체계적 모니터링 및 개입

### 1.4 대상 사용자

#### 1.4.1 주 사용자 (Primary Users)
- **고립은둔청년 (만 19-39세)**
  - 사회적 관계 단절 또는 축소 경험자
  - 우울감, 무력감 등 정신건강 위험군
  - 복지 정보 접근이 어려운 청년층

#### 1.4.2 이차 사용자 (Secondary Users)
- **복지 담당 공무원**
  - 서울시 청년정책 담당자
  - 지역 복지센터 상담사
- **정신건강 전문가**
  - 심리상담사
  - 사회복지사
- **가족 및 보호자**

### 1.5 성공 지표 (KPI)

#### 1.5.1 정량적 지표
- 일일 활성 사용자 수 (DAU): 1,000명 이상 (3개월 내)
- 평균 대화 지속 시간: 15분 이상
- 복지 연계 성공률: 30% 이상
- Emotion Score 기반 위험군 조기 발견율: 70% 이상

#### 1.5.2 정성적 지표
- 사용자 만족도 점수: 4.0/5.0 이상
- 정서적 안정감 개선도
- 복지 접근성 향상도
- 전문가 평가 긍정률

## 2. 기능 요구사항

### 2.1 핵심 기능 (Core Features)

#### 2.1.1 AI 주도형 대화 시스템
**설명**: AI가 능동적으로 사용자에게 먼저 안부를 묻고 대화를 이끌어가는 시스템

**세부 요구사항**:
- 시간대별 맞춤형 인사말 생성
- 이전 대화 맥락 기반 연속성 있는 대화
- 자연스러운 화제 전환 및 질문 생성
- 무응답 시 재참여 유도 메시지 전송

**우선순위**: P0 (필수)

#### 2.1.2 감정 분석 및 Emotion Score 시스템
**설명**: 대화 내용을 실시간 분석하여 감정 상태를 정량화하고 위험도를 평가

**세부 요구사항**:
- 텍스트 기반 감정 분류 (7가지 기본 감정 + 복합 감정)
- 0-100점 스케일의 Emotion Score 산출
- 시계열 기반 감정 변화 추적
- 위험 임계값 도달 시 자동 알림

**우선순위**: P0 (필수)

#### 2.1.3 RAG 기반 복지 정보 제공
**설명**: 서울시 복지 데이터베이스와 연동하여 개인 맞춤형 정책 정보 제공

**세부 요구사항**:
- 실시간 복지 정책 데이터 크롤링 및 업데이트
- 벡터 데이터베이스 기반 의미 검색
- 사용자 프로필 기반 맞춤형 추천
- 정책 신청 방법 및 절차 안내

**우선순위**: P0 (필수)

#### 2.1.4 전문가 연계 시스템
**설명**: 위험군 사용자를 전문가와 자동 연결하는 중개 시스템

**세부 요구사항**:
- Emotion Score 기반 자동 연계 트리거
- 전문가 매칭 알고리즘
- 실시간 알림 및 대응 요청
- 상담 예약 및 일정 관리

**우선순위**: P0 (필수)

### 2.2 사용자 인터페이스 요구사항

#### 2.2.1 모바일 앱 UI/UX
**대상**: 고립은둔청년 (주 사용자)

**디자인 원칙**:
- **접근성 최우선**: 큰 글씨, 명확한 아이콘, 간단한 네비게이션
- **정서적 안정감**: 따뜻한 색상, 부드러운 애니메이션
- **최소 인지 부담**: 한 화면에 하나의 주요 기능
- **반응형 디자인**: 다양한 모바일 기기 지원

**주요 화면 구성**:
- 스플래시 화면
- 온보딩 화면 (3단계)
- 메인 대화 화면
- 정책 정보 카드 화면
- 감정 기록 화면
- 설정 화면

#### 2.2.2 관리자 대시보드
**대상**: 복지 담당자, 전문가

**디자인 원칙**:
- **데이터 시각화**: 직관적인 차트와 그래프
- **실시간 모니터링**: 라이브 업데이트
- **효율적 정보 계층**: 중요도별 정보 배치
- **반응형 웹 디자인**: PC, 태블릿 지원

**주요 화면 구성**:
- 대시보드 메인 (통계 요약)
- 사용자 목록 및 검색
- 개별 사용자 상세 페이지
- 감정 분석 리포트
- 복지 연계 현황
- 알림 및 액션 센터

### 2.3 비기능 요구사항

#### 2.3.1 성능 요구사항
- **응답 시간**: API 호출 평균 200ms 이내
- **동시 접속**: 최소 10,000명 동시 처리
- **가용성**: 99.5% 이상 (월간)
- **데이터 처리**: 일일 100만 건 대화 처리

#### 2.3.2 보안 요구사항
- **데이터 암호화**: AES-256 암호화 (전송/저장)
- **개인정보보호**: GDPR/개인정보보호법 준수
- **접근 제어**: OAuth 2.0 기반 인증
- **감사 로그**: 모든 접근 및 변경 이력 기록

#### 2.3.3 호환성 요구사항
- **모바일 OS**: Android 7.0+, iOS 12.0+
- **웹 브라우저**: Chrome, Safari, Firefox, Edge 최신 2개 버전
- **백엔드 통합**: RESTful API, GraphQL 지원

## 3. 기술 아키텍처

### 3.1 시스템 아키텍처 개요

#### 3.1.1 전체 시스템 구성
시스템은 크게 5개의 레이어로 구성:
1. **프레젠테이션 레이어**: 모바일 앱, 웹 대시보드
2. **API 게이트웨이 레이어**: 요청 라우팅, 인증/인가
3. **비즈니스 로직 레이어**: 핵심 서비스, AI 모델
4. **데이터 레이어**: 데이터베이스, 캐시, 벡터 DB
5. **외부 연동 레이어**: 서울시 API, AI 서비스

### 3.2 프론트엔드 아키텍처

#### 3.2.1 모바일 앱 기술 스택
- **프레임워크**: React Native (크로스 플랫폼)
- **상태 관리**: Redux Toolkit
- **UI 컴포넌트**: React Native Elements
- **네비게이션**: React Navigation
- **실시간 통신**: Socket.io Client
- **로컬 저장소**: AsyncStorage, SQLite

#### 3.2.2 관리자 대시보드 기술 스택
- **프레임워크**: Next.js 14 (React 기반)
- **상태 관리**: Zustand
- **UI 프레임워크**: Material-UI v5
- **차트 라이브러리**: Recharts, D3.js
- **실시간 업데이트**: WebSocket
- **폼 처리**: React Hook Form

### 3.3 백엔드 아키텍처

#### 3.3.1 API 서버
- **프레임워크**: Node.js + Express.js
- **타입스크립트**: 타입 안정성 확보
- **API 문서화**: Swagger/OpenAPI
- **인증/인가**: JWT + Refresh Token
- **미들웨어**: Rate Limiting, CORS, Compression

#### 3.3.2 AI 서비스 레이어
- **주 언어 모델**: Azure GPT-5 API
- **감정 분석 모델**: KoBERT 파인튜닝
- **벡터 임베딩**: OpenAI Embedding API
- **모델 서빙**: FastAPI + Uvicorn
- **모델 관리**: MLflow

#### 3.3.3 워크플로우 자동화
- **오케스트레이션**: n8n (노드 기반 자동화)
- **메시지 큐**: RabbitMQ
- **스케줄링**: node-cron
- **이벤트 처리**: EventEmitter

### 3.4 데이터 아키텍처

#### 3.4.1 주 데이터베이스
- **RDBMS**: PostgreSQL 15
  - 사용자 정보
  - 대화 이력
  - 복지 정책 메타데이터
  - 전문가 정보

#### 3.4.2 벡터 데이터베이스
- **벡터 DB**: Pinecone
  - 정책 문서 임베딩
  - 의미 기반 검색
  - 유사 대화 패턴 매칭

#### 3.4.3 캐싱 레이어
- **Redis**: 세션 관리, 임시 데이터
- **CDN**: 정적 자원 캐싱

#### 3.4.4 데이터 파이프라인
- **ETL**: Apache Airflow
- **데이터 수집**: Python 크롤러
- **데이터 정제**: Pandas, NumPy

### 3.5 인프라 아키텍처

#### 3.5.1 클라우드 플랫폼
- **주 플랫폼**: AWS
  - EC2: 컴퓨팅
  - RDS: 데이터베이스
  - S3: 스토리지
  - CloudFront: CDN
  - Lambda: 서버리스 함수

#### 3.5.2 컨테이너화 및 오케스트레이션
- **컨테이너**: Docker
- **오케스트레이션**: Kubernetes (EKS)
- **레지스트리**: Amazon ECR

#### 3.5.3 모니터링 및 로깅
- **APM**: DataDog
- **로그 수집**: ELK Stack
- **알림**: PagerDuty
